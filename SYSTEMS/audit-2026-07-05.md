## 10. Recommendations

### 🔴 Do These First — Security

1. **Move Docker ports to localhost-only** — Edit `docker-compose.yml`: change `ports: "5432:5432"` → `"127.0.0.1:5432:5432"` for postgres, redis, qdrant, and n8n. Only the services that need internet access (like the website) should be exposed.

2. **Set up fail2ban** — Protects SSH from brute-force:
   ```bash
   sudo apt install fail2ban
   sudo systemctl enable --now fail2ban
   ```

3. **Add Redis password** — Redis needs `requirepass` or use `docker compose` to set `redis-server --requirepass <password>`

4. **Lock down n8n** — Add an N8N_API_KEY or put n8n behind an auth proxy. It's currently wide open on 5678.

5. **Check .gitignore for plugin binaries** — The `.obsidian/plugins/*/main.js` and `.obsidian/plugins/*/styles.css` should be in `.gitignore` (already added, but already tracked — need `git rm --cached` to untrack them).

### 🟠 This Week

6. **Clean up laptop disk** — 83% full with 85 GB remaining. Run:
   ```bash
   sudo apt autoremove && sudo journalctl --vacuum-time=7d
   ```
   Check what's eating space with `ncdu /`.

7. **Install Go on server** — Several tools (arsenal-ng, subfinder, etc.) could benefit. `apt install golang-go`.

8. **Production build website** — Replace `vite dev` with `vite build` + static server. Dev mode has higher memory usage and extra overhead. Create a systemd service for the static file server.

9. **Fix broken cron jobs** — `daily-log` and `morning-briefing` are erroring out. Likely obsidian skill issues. Debug or disable since `nightly-logs` and `mental-health-check` cover similar ground.

### 🟢 Nice to Have

10. **Unify brand** — The landing page uses purple (#6c5ce7) but the main site uses amber (#f59e0b). Pick one and stick with it.

11. **Fix landing page email** — `hello@buildabot.ca` needs a mailbox or redirect if it's supposed to receive inquiries.

12. **Add healthchecks to Docker** — Docker compose supports `healthcheck:` sections for every service. Would give WTCM watchdog richer data.

13. **Enable llama.cpp on laptop startup** — Create a systemd service so the uncensored model auto-starts on boot:
    ```bash
    [Unit]
    Description=llama.cpp server
    After=network.target
    [Service]
    ExecStart=/home/dysthemix/llama.cpp/llama-server -m /path/to/model.gguf --host 0.0.0.0 --port 9000
    Restart=always
    User=dysthemix
    [Install]
    WantedBy=default.target
    ```

14. **Set up automatic git backup** — A cron that pushes the vault changes daily (already handled by `nightly-logs` and `vault-watcher`, but should also push from the laptop side)

15. **Swap to Go for model router** — The Python model router works but a Go binary would be faster, smaller, and more reliable for the chat proxy path. Overkill for now, worth noting.

16. **Add Laptop LXC/KVM for isolated services** — Running everything on bare-metal Parrot OS is fine, but consider Docker on the laptop for services you want isolated.

17. **Enable HTTPS for internal services** — If you ever expose n8n or Qdrant outside Tailscale directly, they need TLS.
